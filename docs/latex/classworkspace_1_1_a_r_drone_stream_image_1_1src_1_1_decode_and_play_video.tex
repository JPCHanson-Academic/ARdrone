We could just display the images as quickly as we decode them, but it turns out we can decode a lot faster than you think.

So instead, the following code does a poor-\/man\textquotesingle{}s version of trying to match up the frame-\/rate requested for each I\+Video\+Picture with the system clock time on your computer.

Remember that all Xuggler I\+Audio\+Samples and I\+Video\+Picture objects always give timestamps in Microseconds, relative to the first decoded item. If instead you used the packet timestamps, they can be in different units depending on your I\+Container, and I\+Stream and things can get hairy quickly.
